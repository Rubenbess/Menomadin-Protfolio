## FIXED ISSUES

### [F-001] CRITICAL: LP report and reminder cron routes silently broken by RLS tightening
- Files: `app/api/cron/lp-report/route.ts`, `app/api/cron/reminders/route.ts`
- Problem: Both routes used `createServerSupabaseClient()` (anon-context). After `rls_tighten_team_members.sql` was applied, every core table requires `exists(select 1 from team_members where id = auth.uid())`. Cron jobs have no user session, so `auth.uid()` is NULL — RLS rejected all reads and the routes returned empty data.
- Fix applied: Switched both routes to `createClient(NEXT_PUBLIC_SUPABASE_URL!, SUPABASE_SERVICE_ROLE_KEY!)` so RLS is bypassed once `requireCronAuth` (bearer secret) succeeds. Added 503 guards for missing Supabase env vars matching existing missing-env handling, and a docstring at each call site explaining the security perimeter.

- Problem: On SAFE conversion the auto-created cap-table row used `'${investor_name} (SAFE)'` (or `'Menomadin (SAFE)'`). Added two tests pinning holder-name values for the Menomadin and external-investor paths so a future regression is caught.
